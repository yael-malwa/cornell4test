Record & replay mock server

Record API Interaction

An HTTP proxy mode in which all interactions are recorded for future use, this is later on saved into "cassetes" for future use.

Replay for Testing

Use the library of "cassetes" to reply pre-recorded responses istead of hitting a real API end point. Save time on testing cycles, avoid flakiness while providing consistent response.

Extend Cornell

Cornell was designed with flexbility in mind such that it should be easy to customize usage beyond the default behaviors.
